# Índice de Entregáveis do Analista - CarWash

## 1. Objetivo

Este índice organiza os entregáveis associados ao papel de Analista de Projetos, sem substituir os documentos base já existentes no projeto.

Os arquivos listados a seguir foram estruturados para facilitar consulta, apresentação e composição da entrega final da disciplina.

---

## 2. Entregáveis complementares

| Entregável | Arquivo | Finalidade |
|---|---|---|
| Documento de Escopo | `documento-de-escopo-funcional-complementar.md` | Consolidar escopo funcional, limites, premissas e critérios do MVP |
| Estudo de Viabilidade | `estudo-de-viabilidade-consolidado.md` | Consolidar viabilidade técnica, econômica, prazo e riscos |
| Backlog detalhado | `backlog-detalhado-do-projeto.md` | Organizar épicos, histórias, prioridades e dependências |
| Endpoints e regras de negócio | `definicao-de-endpoints-e-regras-de-negocio.md` | Relacionar fluxos de negócio com os endpoints da API |
| Plano de Testes | `plano-de-testes-mvp.md` | Definir estratégia, escopo, cenários e evidências de teste |
| Relatório de Bugs | `relatorio-de-bugs-template.md` | Padronizar registro e acompanhamento de defeitos |
| README do projeto | `README.md` | Apresentar o projeto e orientar leitura dos artefatos |

## 4. Regra de uso

- Os documentos complementares devem ser lidos em conjunto com os documentos base.
- Nenhum documento base foi substituído por este pacote complementar.
- Sempre que houver necessidade de rastreabilidade, deve-se priorizar a referência cruzada entre `dvp-e.md`, `drp.md`, `dvs.md`, `4-dat.md`, `5-gdr.md` e os documentos complementares.
